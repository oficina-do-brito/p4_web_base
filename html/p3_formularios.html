Um formulario é feito usando a tag <form>

inputs de login da app 
        
            <form action="para que url vai ser mandado" method="POST " >
                <fieldset> <!-- Os campos de formularios são agupados por fildsets e colocando uma legenda -->
                    <legend>inputs de login da app</legend>
                    <input type="email" name="eml" id="em">
                    <input type="password" name="pass" id="ps">
                    <button type="submit">Enviar</button>
                </fieldset>
            </form>

            Obs importante todo form deve ter seus campos nomeados para serem referenciados no backend atraves desse nome então a propriedade de cada input é importanticimo, 
             *todo formulario deve ter 1 botão de submição
             *todo formulario deve ter o metodo de enviar POST ou GET - diferença entre os métodos GET e POST é no envio de parâmetros. 
             *Isso acontece justamente por que o protocolo HTTP define que cada método tem funções diferentes. O GET é usado geralmente 
            para “pegar” informações (fazer filtros, querys) e o método POST é usado para “postar” (enviar pra um backend) informações.
